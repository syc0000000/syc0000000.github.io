安装hexo下载环境

  1. 设置淘宝镜像下载

    npm install -g cnpm --registry=https://registry.npm.taobao.org

  2. 安装cnpm

    cnpm

本地配置hexo

  1. 安装hexo

    cnpm install -g hexo -cli

  2. 初始化hexo

    hexo init

报错: (没有保留到截图,根据搜索记录还原)

本地测试blog

    hexo s

  3. 访问localhost:40000

将博客配置到GitHub

  1. 根据教程:

报错:

Error: Spawn failed

f 搜 搜索后, 找到方案二: Hexo部署出现错误 Error: Spawn failed 解决方式 | 毋忘草的小岛 (myosotis.xin)

更改后:

不再报错

  1. 安装部署插件

    npm install --save hexo-deployer-git

  2. 部署更新

    hexo clean （清除缓存）

    hexo g 即(hexo generate)

    hexo d 即(hexo deploy)

报错:

(具体忘了,大概就是)Time out

猜测是因为 GFW 干扰, 所以科学上网后解决.

  1. 访问测试

错误:

页面只有一个标题: syc0000000.github.io

经过自己的研究, 发现是 GitHub pages 默认使用 main 分枝, 而按照教程,上图中 branch 填写的是 master .所以GitHub上自动创建了一个名为 master 的分枝. 于是我在 GitHub pages 页面将分枝改为 master .等待一两分钟后, 正常访问.

至此, 博客搭建完成
