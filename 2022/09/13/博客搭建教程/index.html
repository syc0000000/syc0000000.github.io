博客搭建教程

离谱的博客搭建过程..

宋雨宸

搭建的时候没看到要写blog, 所以是后来补的, 没有多少 Git bash 的图片, 见谅. 如果有幸参与面试, 如果要了解可以问.

耗时7小时(18点-1点), 麻了. (虽然同时还完成了request)

寻找教程

根据网络资料(Hexo+Github Pages快速打造属于自己的网站 | royhuang’s blog (hwh.zone)),决定使用 hexo + GitHub pages 的方案

参考题目中推荐的教程, 发现GitHub界面已经改版几次, hexo和node.js版本过于古老, 可能存在安全性问题. 所以在 f 搜上限定时间一年内,找到如下教程

https://www.jianshu.com/p/2fc155218a6f

(不该照这个, 有一个错…)

辅助资料:

配置环境

git在basic栏目就已经装好, 只需配置.

错误

教程少打了一个”-“

git config -–global user.name “2869993866@qq.com”

node.js已经按照题目推荐教程安装,就没按照这个教程安装. (我不该偷懒的, 见下面……)

github 账号之前已注册.

安装hexo下载环境

  1. 设置淘宝镜像下载

    npm install -g cnpm --registry=https://registry.npm.taobao.org

  2. 安装cnpm

    cnpm

本地配置hexo

  1. 安装hexo

    cnpm install -g hexo -cli

  2. 初始化hexo

    hexo init

报错: (没有保留到截图,根据搜索记录还原)

TypeError [ERR_INVALID_ARG_TYPE]: The “mode” argument must be integer. Received an instance of Object

部署Hexo踩过的坑—node14.0配置hexo - 知乎 (zhihu.com)

于是我降低了node.js版本(我以为是降低, 其实之前根据题目给的教程里面的链接下载的更低)

然后继续报其他错…

再一番搜索后发现是版本过低

How to fix ‘Unsupported platform for fsevents@1.2.9: wanted {"os":"darwin","arch":"any"} (current: {"os":"win32","arch":"x64"}) - fsou (nilmap.com)

具体还有什么报错给忘了, 反正后来就装上了

本地测试blog

  1. 安装插件

    npm install hexo-server --save

  2. 启动本地调试hexo

    hexo s

  3. 访问localhost:40000

​ 正常

配置GitHub ssh

成功:

创建GitHub仓库

  1. 由于我的 GitHub ID 是syc0000000, 所以建立名字叫 syc0000000.github.io 的仓库, 并开通GitHub pages.

将博客配置到GitHub

  1. 根据教程:

报错:

Error: Spawn failed

f 搜 搜索后, 找到方案二: Hexo部署出现错误 Error: Spawn failed 解决方式 | 毋忘草的小岛 (myosotis.xin)

更改后:

不再报错

  1. 安装部署插件

    npm install --save hexo-deployer-git

  2. 部署更新

    hexo clean （清除缓存）

    hexo g 即(hexo generate)

    hexo d 即(hexo deploy)

报错:

(具体忘了,大概就是)Time out

猜测是因为 GFW 干扰, 所以科学上网后解决.

  1. 访问测试

错误:

页面只有一个标题: syc0000000.github.io

经过自己的研究, 发现是 GitHub pages 默认使用 main 分枝, 而按照教程,上图中 branch 填写的是 master .所以GitHub上自动创建了一个名为 master 的分枝. 于是我在 GitHub pages 页面将分枝改为 master .等待一两分钟后, 正常访问.

至此, 博客搭建完成
